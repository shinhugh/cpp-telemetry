# cpp-telemetry

A C++ library that provides tracing and logging.

### Capabilities

- Generate a trace tree that captures the entire hierarchy of operations triggered by a root operation.
- Use RAII to directly tie a span to the lifetime of an operation.
- Produce a structured log that can capture any C++ type.
- Support custom serialization formats and delivery pipelines for logs and spans.
- Utilize batching for span and log publication.

---

## Getting started

### Prerequisites

- Clang must be installed for the provided build script to work.\
Developers are free to use other build systems, but this repository supports only Clang out of the box.

### Building

Navigate to the project's root directory.

Run `build.sh`.

This creates the subdirectory `build/` and populates it with the build artifacts.

### Integration

Use the header files in `telemetry/include/` to add instrumentation to any application, and link the application with the static library archive `libtelemetry.a` generated by `build.sh`.

Applications should be instrumented with spans before logs are introduced. Logging requires tracing to be in place (explained below).

---

## Interface

The logging interface requires every log statement to specify a span.

- A span expresses that an operation happened. It shows the hierarchical and temporal position of the operation within the greater scope of a trace.\
_Spans form the skeleton._
- A log provides visibility into the state of a single operation. It decorates a span with additional details specific to the operation.\
_Logs serve as the flesh that attaches to the skeleton._

With this paradigm, it only makes sense that every log be associated with a span.

### Tracing: Instrumentation

`#include "telemetry/living_span.h"`

---

```c++
telemetry::LivingSpan::LivingSpan(
    bool openOnCreate)
```

Create an object whose lifetime directly maps to the lifetime of a span.\
If `openOnCreate` is `true`, the span's start time is immediately set.\
If `openOnCreate` is `false`, `LivingSpan::Open()` must explicitly be called, at which point the span's start time is set.\
The span's finish time is set by either explicity calling `LivingSpan::Close()` or destroying the object.\
The span is added to the publication queue when the finish time is set.

---

```c++
telemetry::LivingSpan::LivingSpan(
    const std::string &tag, bool openOnCreate)
```

Create an object whose lifetime directly maps to the lifetime of a span.\
This is a variation of `LivingSpan::LivingSpan(bool openOnCreate)` that allows for a tag to be specified. A tag is simply a string that is added to the span ID (and the trace ID because this is a root span) to make it easier to distinguish the span (and trace).

---

```c++
telemetry::LivingSpan::LivingSpan(
    std::string &&tag, bool openOnCreate)
```

Create an object whose lifetime directly maps to the lifetime of a span.\
This is a variation of `LivingSpan::LivingSpan(const std::string &tag, bool openOnCreate)` that accepts an rvalue for `tag`.

---

```c++
telemetry::LivingSpan::LivingSpan(
    const char *tag, bool openOnCreate)
```

Create an object whose lifetime directly maps to the lifetime of a span.\
This is a variation of `LivingSpan::LivingSpan(const std::string &tag, bool openOnCreate)` that accepts a C-string for `tag`.

---

```c++
telemetry::LivingSpan::LivingSpan(
    const telemetry::Span &parentSpan, bool openOnCreate)
```

Create an object whose lifetime directly maps to the lifetime of a span.\
This is a variation of `LivingSpan::LivingSpan(bool openOnCreate)` that allows for a parent span to be specified. The created span inherits the trace ID from the parent span and sets its own parent span ID as the parent span's span ID.

---

```c++

    const telemetry::Span &parentSpan, const std::string &tag,
    bool openOnCreate)
```

This is a variation of `LivingSpan::LivingSpan(const Span &parentSpan, bool openOnCreate)` that allows for a tag to be specified. A tag is simply a string that is added to the span ID to make it easier to distinguish the span.

---

```c++
telemetry::LivingSpan::LivingSpan(
    const telemetry::Span &parentSpan, std::string &&tag, bool openOnCreate)
```

Create an object whose lifetime directly maps to the lifetime of a span.\
This is a variation of `LivingSpan::LivingSpan(const Span &parentSpan, const std::string &tag, bool openOnCreate)` that accepts an rvalue for `tag`.

---

```c++
telemetry::LivingSpan::LivingSpan(
    const telemetry::Span &parentSpan, const char *tag, bool openOnCreate)
```

Create an object whose lifetime directly maps to the lifetime of a span.\
This is a variation of `LivingSpan::LivingSpan(const Span &parentSpan, const std::string &tag, bool openOnCreate)` that accepts a C-string for `tag`.

---

```c++
telemetry::LivingSpan::Open()
```

Explicitly set a span's start time to the current time if `openOnCreate` was `false` when the `LivingSpan` constructor was called.

---

```c++
telemetry::LivingSpan::Open(
    std::chrono::steady_clock::time_point)
```

Explicitly set a span's start time to the specified time if `openOnCreate` was `false` when the `LivingSpan` constructor was called.

---

```c++
telemetry::LivingSpan::Close()
```

Explicitly set a span's finish time to the current time and add the span to the publication queue.\
Destroying the `LivingSpan` will not duplicate this process.

---

```c++
telemetry::LivingSpan::Close(
    std::chrono::steady_clock::time_point)
```

Explicitly set a span's finish time to the specified time and add the span to the publication queue.\
Destroying the `LivingSpan` will not duplicate this process.
